# MSP 고객 DB 마이그레이션 매핑

안녕하세요, 동식님! 🙇
기존 노션 MSP 고객 DB(57개 고객)의 각 컬럼이 새로운 CRM의 어떤 필드로 옮겨갈지 정리해보았습니다. 확인 부탁드립니다!

CRM에서 MSP 고객 정보는 **고객 단위**와 **계약 단위**로 나뉘어 저장됩니다.

- **고객 단위**에 저장: 고객명, 산업, 회사 규모, 상태 등 고객 고유의 정보
- **계약 단위**에 저장: MSP 등급, 크레딧 쉐어, AWS 담당 AM, 청구 방식, 태그 등 계약별로 달라지는 정보
- **담당자(연락처)**: 고객사의 담당자 이름/이메일/전화
- **담당 기술(다중)**: 한 계약에 여러 명의 기술 담당자를 연결

---

## 📋 입력 형식 설명 (범례)

표에서 사용하는 형식 용어를 먼저 정리해두었습니다.

| 표기 | 의미 |
|---|---|
| **자유 입력** | 아무 글자나 입력 가능 |
| **선택지: A / B / C** | 정해진 항목 중 하나만 선택 |
| **여러 값** | 한 칸에 여러 항목을 저장 (쉼표로 구분) |
| **Yes / No** | 등록/미등록, 있음/없음처럼 두 가지 중 하나 |
| **날짜 / 날짜·시간** | 달력에서 고르는 값 |
| **사내 직원 선택** | 직원 목록에서 선택 (이름은 시스템이 ID로 변환) |

## ❓ 확인 필요

아래 항목들은 일단 매핑은 해두었으나, 세부 옵션이나 의미를 한 번만 확인 부탁드립니다 🙏

| 노션 컬럼 | 예시 | 잠정 매핑 | 현재 CRM 선택지 | 확인 필요 사항 |
|---|---|---|---|---|
| 산업군 | IT, 공공 (+ 빈값 다수) | 고객 정보 > **산업분야** | IT / 제조 / 금융 / 유통 / 공공 / 서울대 연구실 / 기타 | `서울대 연구실` 선택지 유지 여부 |
| 청구 방식 | (58개 모두 빈값) | 계약 > MSP 상세 > **청구 방식** | 대표님 직접 청구 / 매월 10일 세금계산서 발행 / 공공기관 별도 청구 | 현재 선택지가 적절한지, 추가·변경할 항목이 있는지 |
| EUR/빌링온 | Yes / No | 계약 > MSP 상세 > **빌링온** | Yes / No | "빌링온"의 정확한 의미 및 저장 방식 확인 |

### 산업군 `서울대 연구실` 옵션
- 노션 원본에 있어서 CRM 선택지에도 그대로 포함해두었습니다.
- CRM에서도 이 옵션을 유지할 필요가 있을지 확인 부탁드립니다.
  - 유지 시: `IT / 제조 / 금융 / 유통 / 공공 / 서울대 연구실 / 기타`
  - 제거 시: `IT / 제조 / 금융 / 유통 / 공공 / 기타` (서울대 연구실은 `기타`에 통합)

### 청구 방식 선택지
- 노션 원본 옵션을 그대로 옮겨온 상태입니다.
- 현재 선택지: `대표님 직접 청구 / 매월 10일 세금계산서 발행 / 공공기관 별도 청구`
- 그대로 사용해도 괜찮을지, 추가·변경이 필요한 항목이 있으면 말씀해주세요!
- (노션에서 58개 모두 빈값이라 실제 사용 케이스를 파악하기 어려워 여쭤봅니다 🙏)

### EUR/빌링온 (Yes / No)
- 정확히 어떤 의미인지 알려주시면 감사하겠습니다!
- CRM에는 계약 > MSP 상세 > **빌링온**에 Yes/No 토글로 저장할 예정인데, 이대로 괜찮을까요?

<details>
<summary>📎 참고: CRM DB 구조</summary>

### clients (고객)
- `id`, `client_id` (자동 생성)
- `name`, `client_type` (univ/corp/govt/asso/etc)
- `grade` (A/B/C/D/E), `business_types`, `parent_id`
- `status` (신규/진행중/활성/휴면/종료/상태없음)
- `memo`

### client_msp_details
- `client_id` (FK)
- `industry`, `company_size` (enum)
- `tags` (text[])
- `memo`

### contracts (계약)
- `id`, `contract_id` (자동)
- `client_id`, `type` (msp), `name`
- `total_amount`, `currency`, `stage`
- `memo`, `assigned_to`

### contract_msp_details
- `contract_id` (FK)
- `msp_grade`, `credit_share`, `payer`, `billing_method` (enum)
- `sales_rep_id` (FK → employees)
- `aws_am`, `aws_account_ids` (text[]), `aws_amount`
- `expected_mrr`, `has_management_fee`
- `billing_on` (bool), `billing_on_alias` (text)

### contacts (담당자)
- `client_id` (FK)
- `name`, `email`, `phone`, `department`, `position`, `role`
- `is_primary`

### contract_tech_leads (담당 기술 — 다중)
- `contract_id` (FK), `employee_id` (FK → employees)
- PK: `(contract_id, employee_id)`

</details>
